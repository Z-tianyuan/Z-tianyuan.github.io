你问出了一个渗透测试工程师走向成熟的**终极问题**。

照着教程把漏洞复现出来，那叫“漏洞验证”；但在一个完全未知的、没有任何提示的黑色盒子里把漏洞揪出来，那才叫真正的**“漏洞挖掘 (Bug Hunting)”**。

当你以后在真实的漏洞赏金平台（比如 HackerOne 上的企业级目标）进行实战挖掘时，你是绝对碰不到这种给你写好前提条件和步骤的靶场的。面对一个复杂的企业级系统，黑客寻找这类**“状态机逻辑缺陷 (Flawed State Machine)”**漏洞，有一套极其严密的标准流程（SOP）。

这是一套被称为**“业务流逆向降维分析”**的实战 WBS。我为你总结了四个阶段：

### 🕵️‍♂️ 状态机漏洞发现 SOP (标准作业程序)

#### 第一阶段：全量状态测绘 (Mapping the Happy Path)

不要一上来就想着搞破坏！黑客的第一步是把自己伪装成一个极其听话的“好用户”。

1. **清空历史：** 在 Burp 的 HTTP History 中打一个标记（或者清空之前的记录），确保接下来的流量是纯净的。

2. **顺流而下：** 关闭拦截 (Intercept is off)。在浏览器中，**极其缓慢且完整地**走完整个业务流程（比如：输入账号密码 -> 点击登录 -> 页面跳转 -> 选择角色 -> 进入主页）。

3. **绘制拓扑图：** 回到 Burp，把刚才这套流程产生的所有核心 URL 提取出来，在脑子里（或纸上）画一条线： `A (/login)` ➡️ `B (/role-selector)` ➡️ `C (/home)`

#### 第二阶段：锚定“权力交接点” (Identify State Transitions)

这是最考验深度思考的一步。你需要看着 HTTP History 里的数据包，找出**系统是在哪一刻决定你的权限的**。

1. **追踪 Cookie/Token 的诞生：** 仔细看 `A (/login)` 的响应包。它是不是在这里通过 `Set-Cookie` 给你发了身份证？

2. **追踪权限的降级/确认：** 仔细看 `B (/role-selector)` 的请求和响应。系统是不是在这个环节，把你刚才拿到的通用身份证，打上了“普通员工”的钢印？

#### 第三阶段：实施“强制阻断”与“孤岛测试” (The Drop & Skip)

一旦你发现权力的授予（A）和权力的限制（B）是分两步走的，**巨大的漏洞机会就出现了**。
开发者往往会默认“用户一定会乖乖从 A 走到 B”。你的任务就是打破这个默认设定。

1. **开启拦截模式 (Intercept is on)。**

2. **重走流程，并在关键点拔刀：** 再次提交登录请求（A）。当服务器返回成功，并试图让你的浏览器自动跳转去（B）接受权限限制时，**在 Burp 里果断点击 `Drop`**。

3. **孤岛状态评估：** 此时，你的浏览器停留在了一个“半路出家”的孤岛状态。你手里拿着 A 阶段发给你的、还没来得及被 B 阶段降级的“原生 Token”。

#### 第四阶段：盲测高权限接口 (Blind Authorization Testing)

1. **直接突袭：** 关闭拦截。带着你这把“原生 Token”，直接在浏览器地址栏强行输入最高权限的路径（比如 `/admin`、`/console`、`/delete` 等你在第一阶段测绘时可能注意到的，或者靠字典猜出来的敏感路径）。

2. **确认战果：** 如果服务器放行了，说明你成功卡出了一个“初始高权限状态”。漏洞挖掘成功！
