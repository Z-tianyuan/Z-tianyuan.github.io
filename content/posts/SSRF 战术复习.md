#### 第一层境界：精准定位靶心 (目标在哪？)

- **对本地服务器进行基本 SSRF：** 你学会了让外网服务器攻击它自己。目标锁定：`http://localhost/admin` 或 `127.0.0.1`。
    
- **基础 SSRF 对抗另一后端系统：** 你学会了利用外网服务器作为跳板，去扫描整个内网盲区。你甚至为此亲手写了一个 Python 多线程并发扫描器，找出了隐藏在 `192.168.0.x:8080` 的内网管理面板。
    

#### 第二层境界：千变万化的伪装 (如何骗过保安？)

- **黑名单绕过 (死板的违禁词)：** 当防御者禁用了 `127.0.0.1` 和 `admin`。你学会了利用操作系统的自动补全（`127.1`）绕过 IP 检测，又利用你自己写的 Python 工具生成了 `%2561dmin`（双重 URL 编码），利用后端的多次解码机制骗过了安全器。
    
- **白名单绕过 (解析器认知分裂)：** 这是最惊艳的底层魔法！你利用了 `@` 符号（凭证注入）满足了安全器对 `stock.weliketoshop.net` 的白名单校验，同时利用了 `%2523` (`#` 锚点) 截断了底层 HTTP 客户端的请求路径，成功将车开进了 `localhost`。
    
- **开放重定向绕过 (借刀杀人)：** 当安全器极度严格，只允许相对路径（`/product/`）时。你没有硬碰硬，而是找到了网站上另一个存在“开放重定向”的合法接口（`/product/nextProduct?path=`），利用 HTTP 客户端自动跟随跳转的特性，在合法掩护下完成了内网穿越。
