#### 📍 第一步：寻找“代购”入口 (Locate the Sink)

你要寻找一切**让服务器去发起网络请求**的功能点。常见的重灾区有：

- **URL 传参点：** 像刚才的 `stockApi=`，或者 `url=`, `domain=`, `endpoint=`。
    
- **图片加载/头像上传：** “提供图片 URL 自动下载并设置为头像”的功能。
    
- **PDF 导出器：** 将网页转换成 PDF 的功能（很多 HTML 转 PDF 的引擎允许内嵌远程资源）。
    
- **Webhooks 订阅：** 允许你输入一个回调地址，服务器有事会向这个地址发请求。
    

#### 📍 第二步：回显探针 (Proof of Concept)

找到入口后，不要急着打内网，先证明它真的能发请求。

- **测试方法：** 把参数改成你自己控制的公网服务器地址（比如你自己买的云服务器 IP，或者使用 Burp Collaborator 提供的临时域名）。
    
- **判定：** 如果你的服务器收到了来自目标机器的 HTTP 请求或 DNS 查询，恭喜你，盲打成功，存在 SSRF 漏洞！
    

#### 📍 第三步：探测内网轮廓 (Internal Reconnaissance)

这就是你刚才用 Python 脚本做的事情。一旦确认 SSRF 存在，立刻开始“扫街”。

- **猜网段：** 尝试常见的内网私有地址段，如 `192.168.x.x`, `10.x.x.x`, `172.16.x.x`。
    
- **探本机：** 永远不要忘记测试 `127.0.0.1` 和 `localhost`，看看服务器本机有没有藏着未授权的后台（就是你的第一个 SSRF 实验）。
    
- **扫端口：** 用 Intruder 或 Python 并发扫描，重点关注 `80, 443, 8080` (Web服务), `22` (SSH), `6379` (Redis数据库)。
    

#### 📍 第四步：发动致命打击 (Exploitation)

根据你扫出来的内网服务，实施定点爆破。

- **打脆弱的 Web 后台：** 就像你刚才做的，拼接 `/admin/delete?username=carlos`。
    
- **读取本地文件：** 这一步极其致命！把 `http://` 换成 `file://` 协议。比如测试 `file:///etc/passwd` (Linux) 或 `file:///c:/windows/win.ini` (Windows)。如果成功，你就能直接读取服务器硬盘上的任何文件！


### 💡 避坑指南：你刚才踩过的雷

把这两句话刻在你的备忘录上：

1. **端口就是门牌号，绝对不能丢！** 盲扫出 IP 只是找到了大楼，不带端口（比如 `:8080`）就找不到房间。
    
2. **在 POST 的 Body 里手动改参数，必须 URL 编码！** 不要让特殊字符（`:` `/` `?` `=`）破坏了 HTTP 协议的原本结构。遇到复杂的 Payload，果断 `Ctrl + U`。
