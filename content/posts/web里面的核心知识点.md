#### 1. `@` 符号：被遗忘的“账号密码”分隔符

你平时看到的 URL 都是 `http://www.baidu.com`。 但其实，HTTP 协议允许你在 URL 里直接带上账号密码，格式是： `http://账号:密码@目标网站.com` 例如：`http://admin:123456@stock.weliketoshop.net`。 在这个格式里，**`@` 符号后面的才是真正的目标主机地址。**

#### 2. `#` 符号：只给浏览器看的“书签” (Fragment)

在网址最后加上 `#`，比如 `http://example.com/page.html#section1`，这叫 URL 锚点/片段。 关键特性：**`#` 后面的内容，通常只在前端（浏览器）起作用，后端的网络请求库（比如发起 HTTP 请求的代码）一看到 `#`，就会自动丢弃后面所有的内容！**

### 🎭 核心概念：什么是重定向 (Redirect)？

在网络世界里，重定向就像是现实中的**“邮件转发”**或者**“搬家告示”**。

1. 你去访问网址 A。
    
2. 网址 A 的服务器回给你一个特殊的状态码 **`302 Found`**，并在回信里带上一个 **`Location: 网址 B`** 的头部。
    
3. 你的浏览器（或者后端的 HTTP 客户端代码）一看到这个 `302` 和 `Location`，就会**非常听话地、自动地**去请求网址 B。
    

**什么是“开放重定向”漏洞？** 如果网址 A 跳转到哪里的这个决定权，是可以通过参数被用户控制的（比如 `/product/nextProduct?path=你要跳转的网址`），而且服务器**没有任何验证**就直接跳转过去，这就叫开放重定向。单独看，它最多只能用来钓鱼；但一旦和 SSRF 结合，它就成了神级跳板！


### 参数敏感度

 在 Web 安全领域，只要看到参数名叫 `path`, `url`, `redirect`, `to`, `forward` 等等，黑客的眼睛都会冒绿光。
