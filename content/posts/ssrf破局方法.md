在真实的漏洞挖掘（Bug Bounty）中，绝对没有任何人会提前把 `stock.weliketoshop.net` 或者 `localhost:80` 塞到你的手里。黑客在实战中找出这些信息，靠的是**观察**和**扫描**。

我们来复盘一下，在真实的黑盒测试中，你是怎么一步步“榨”出这两个关键信息的。

### 🕵️‍♂️ 破局点一：怎么套出保安的“白名单”（`stock.weliketoshop.net`）？

你觉得这个白名单很难猜？其实它往往就写在你的脸上！

1. **观察正常业务流量（最常见）：** 当你正常点击“检查库存”按钮时，抓下来的第一个原始数据包，它的 `stockApi` 参数里填的是什么？ 它填的肯定是：`http://stock.weliketoshop.net/product/stock/check...` **黑客直觉：** 既然服务器原本就是去访问这个域名的，那么这个域名 99% 就是程序员写死在代码里的“白名单”！
    
2. **故意报错，让服务器“说漏嘴”（Fuzzing）：** 如果你故意输入一个垃圾网址，比如 `stockApi=http://www.baidu.com`。 有些开发者的错误处理写得很烂，服务器会直接弹出一个报错页面，上面赫然写着： `Error: URL must contain stock.weliketoshop.net` 这叫**敏感信息泄露**，白名单直接白给。
    

---

### 🕵️‍♂️ 破局点二：不知道内网 IP 怎么办？（结合前面的绝招！）

你的直觉非常敏锐：在实战中，内网的管理面板绝对不一定在 `localhost`，也不一定在 `80` 端口。它可能在 `192.168.1.155` 的 `8080` 端口。

如果不知道目标在哪，怎么办？**答案是：带着你的“伪装面具”，重新启动内网大扫荡！**

这需要把你刚才学到的两个神技**结合起来**：

1. **你拥有了绕过白名单的万能公式（面具）：** `http://[你的目标]%2523@stock.weliketoshop.net`
    
2. **你拥有了内网并发扫描的技术（雷达）：** 还记得你刚才用 Python 写的那个 `192.168.0.1` 到 `255` 的扫描脚本吗？
    

在真实的实战中，黑客的做法是**把扫描脚本里的 Payload 换成带有绕过面具的版本**！

你会在你的 Python 脚本或者 Burp Intruder 里，这样构造子弹：

- **扫 IP：** `http://192.168.0.§X§:80%2523@stock.weliketoshop.net/admin` （让 X 从 1 变到 255）
    
- **扫端口：** 如果 IP 确定是 localhost，但不知道端口，你就扫端口：`http://localhost:§PORT§%2523@stock.weliketoshop.net/admin` （让 PORT 跑一遍常见的 80, 8080, 8888, 8000）
