#### 📍 步骤 1：侦察与雷达锁定 (寻找“代购”入口)

全站乱点，盯紧 Burp Suite 的 HTTP 历史记录。寻找一切让服务器**主动发起网络请求**的参数和功能点。

- **高危参数名：** `url=`, `api=`, `path=`, `dest=`, `redirect=`, `uri=`, `domain=`。
    
- **高危业务场景：**
    
    - **图片/头像拉取：** “输入 URL 自动下载图片”的功能。
        
    - **Webhooks：** 让你配置回调地址的地方。
        
    - **PDF 导出：** 很多把 HTML 转 PDF 的功能会解析页面里的远程资源。
        
    - **翻译/文章抓取：** 输入链接自动提取正文的功能。
        

#### 📍 步骤 2：盲打探针 (证明它能往外发包)

找到疑似点后，**不要急着打内网！** 先用你自己的公网服务器 IP，或者 Burp Collaborator 提供的临时域名。

- 发送请求：`url=http://你的域名.com`
    
- **判定：** 如果你的服务器收到了来自目标机器的 HTTP 请求或 DNS 解析记录，恭喜你，盲打成功！这就已经是有效的漏洞了。
    

#### 📍 步骤 3：内网雷达大扫荡 (探测内部资产)

确认 SSRF 存在后，开启你的 Python 扫描脚本或 Burp Intruder，对着目标的内网进行轰炸。

- **探本机：** `127.0.0.1`, `localhost`, `0.0.0.0`, `127.1`
    
- **探局域网：** `192.168.x.x`, `10.x.x.x`, `172.16.x.x`
    
- **探端口：** 重点扫描 `80`, `8080`, `8888` (Web后台), `6379` (Redis，极易造成未授权命令执行), `22` (SSH)。
    

#### 📍 步骤 4：防御穿透与绝杀 (遇到拦截时的军火库)

如果被 WAF 或代码拦截，依次掏出你的绕过武器：

1. **IP 变形：** 用 `127.1`、十六进制 IP (`0x7F000001`)、八进制 IP 替换正常的 IP。
    
2. **编码变形：** 用你写的脚本进行双重甚至三重 URL 编码。
    
3. **找内鬼跳板：** 在网站上寻找 `redirect=` 或 `path=` 这种开放重定向漏洞，组合利用。
    
4. **解析器欺骗：** 尝试 `http://127.0.0.1%2523@白名单域名.com`。
